# Planning And Design Workflow

Use this workflow when a new scope starts or when implementation needs fresh decisions before code changes.

## Covers

- Phase 1. Plan
- Phase 2. Design

## Skills To Use

- `skills/planning-agent/SKILL.md`
- `skills/designing-agent/SKILL.md`

## Sequence

1. Start with `skills/planning-agent/SKILL.md` to refresh `.dev/DASHBOARD.md`, `.dev/TASKS.md`, and the next action for the active scope.
2. Move to `skills/designing-agent/SKILL.md` once the phase breakdown is clear.
3. Capture only the design decisions that unblock implementation, test writing, recovery, and validation.
4. Route back to planning if the scope, risks, or milestone boundaries change materially.

## Outputs

- Prompt-derived phase checklist in `.dev/TASKS.md`
- Real in-progress status in `.dev/DASHBOARD.md`
- Implementation-ready design notes and interfaces
